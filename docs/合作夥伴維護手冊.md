## 1. 首頁長什麼樣 ↔ 由哪些表控制

首頁分兩區：**左邊「連結目錄」** + **右邊「公告區」**。四張表分別控制：

```
首頁
├─ 連結目錄（左）
│   ├─ 大分組卡片      ← portal_link_group   （例：表單作業 / 資料查詢與後台）
│   │   ├─ 分類子標題  ← portal_link_category（例：主檔管理 / 電費作業）
│   │   │   └─ 連結項目 ← portal_link         （例：案場管理 → /master/site）
└─ 公告區（右）        ← portal_notice        （例：歡迎詞、置頂公告）
```

### 各欄位 → 畫面對應（範例資料 → 首頁長什麼）

> 以下用「一筆資料 → 首頁長出什麼」示範。看懂一個，其他同理。

#### ① `portal_link_group` — 大分組（一張大卡片）

範例列：

| group_name | sort_order | is_enabled |
|---|---|---|
| 表單作業 | 10 | true |

首頁長出一張大卡片：

```
┌─ 表單作業 ──────────────────────────────┐  ← group_name = 卡片大標題
│  （這個分組的分類與連結都放在這張卡裡）    │
└─────────────────────────────────────────┘
```
- 改 `group_name` → 換卡片大標題
- 改 `sort_order` → 換卡片上下順序（數字小的在上）
- `is_enabled` 設 `false` → **整張卡片從首頁消失**
- `group_code`：內部代碼，分類靠它歸到這張卡，**命名後別再改**

#### ② `portal_link_category` — 分類（卡片內的小標題）

範例列（`group_code=forms` 表示掛在上面「表單作業」那張卡底下）：

| category_name | group_code | sort_order |
|---|---|---|
| 主檔管理 | forms | 10 |

在「表單作業」卡片裡長出一個小標題區：

```
┌─ 表單作業 ──────────────────────────────┐
│  主檔管理                ← category_name = 這個小標題
│    • 案場管理  …（這個分類的連結列在下面）
│    • 電號管理  …
└─────────────────────────────────────────┘
```
- 改 `category_name` → 換小標題
- 改 `group_code` → 把這個分類**搬到別張大卡**（值要等於某個分組的 `group_code`；對不到就不顯示）
- 改 `sort_order` → 換分類在卡片內的順序
- `category_code`：內部代碼，連結靠它歸到這個分類，**別再改**

#### ③ `portal_link` — 連結（實際可點的項目）

範例列（`category_code=master` 表示掛在「主檔管理」分類底下）：

| category_code | title | url | description | is_new |
|---|---|---|---|---|
| master | 案場管理 | /master/site | 案場建立/更新/停用 | true |

長出一條可點的連結：

```
┌────────────────────────────────────────┐
│  案場管理  [新]                  前往 →  │  ← title；[新] 來自 is_new=true
│  案場建立/更新/停用                       │  ← description（下方說明小字）
└────────────────────────────────────────┘
        點一下 → 跳到 /master/site           ← url
```
- 改 `title` → 換顯示名稱
- 改 `url` → 換點了跳去哪（站內頁 `/master/site`；外部網站 `https://...`）
- 改 `description` → 換下方說明小字
- `is_new` 設 `false` → **「新」標籤消失**
- 改 `category_code` → 把連結**搬到別的分類**（值要等於某個分類的 `category_code`）
- `is_enabled` 設 `false` → **這條連結從首頁消失**

#### ④ `portal_notice` — 公告（首頁右側公告區）

範例列：

| title | content | pinned |
|---|---|---|
| 歡迎使用好室包租代管系統 | （空白） | true |

在右側公告區長出：

```
公告區
┌──────────────────────────────────┐
│  歡迎使用好室包租代管系統  [置頂]   │  ← title；[置頂] 來自 pinned=true
│  （若 content 有填，會顯示在這一行）│  ← content（空白就不顯示這行）
└──────────────────────────────────┘
```
- 改 `title` → 換公告標題
- 改 `content` → 換內文（留空 = 只顯示標題）
- `pinned` 設 `true` → **置頂並顯示「置頂」標籤**
- 改 `sort_order` → 換順序（置頂的排在前）
- `is_enabled` 設 `false` → **這則公告從首頁消失**

> 改完 → 回瀏覽器**按 F5 重新整理**首頁，立刻看到變化（不用重部署）。

## 2. 常見維護動作（可直接複製改）

> SQL 在 pgAdmin 的 `haoshi_rental` Query Tool 執行。文字用**單引號** `'...'`，中文沒問題。

### 發一則新公告
```sql
INSERT INTO portal_notice (title, content, pinned, sort_order)
VALUES ('本週五系統維護', '5/10 20:00-21:00 暫停服務', true, 5);
```

### 改公告內容
```sql
UPDATE portal_notice SET content = '改好的新內容' WHERE title = '本週五系統維護';
```

### 暫時隱藏一則公告（保留、之後可再開）
```sql
UPDATE portal_notice SET is_enabled = false WHERE title = '本週五系統維護';
```

### 新增一個連結（放到「主檔管理」分類）
```sql
INSERT INTO portal_link (category_code, title, url, description, is_new, sort_order)
VALUES ('master', '新功能頁', '/some-new-page', '這是新功能', true, 99);
```

### 隱藏 / 改名 / 改排序一個連結
```sql
UPDATE portal_link SET is_enabled = false WHERE title = '稽核紀錄';   -- 隱藏
UPDATE portal_link SET title = '案場資料維護'   WHERE title = '案場管理'; -- 改名
UPDATE portal_link SET sort_order = 5           WHERE title = '房號管理'; -- 提前
```

### 新增一個分類 / 分組
```sql
-- 先建分類（掛在 forms 分組底下）
INSERT INTO portal_link_category (category_code, group_code, category_name, sort_order)
VALUES ('report', 'forms', '報表專區', 30);

-- 或建一個新分組
INSERT INTO portal_link_group (group_code, group_name, sort_order)
VALUES ('tools', '其他工具', 30);
```

### 查目前首頁有哪些內容（對照用）
```sql
SELECT g.group_name, c.category_name, l.title, l.url, l.is_enabled
FROM portal_link l
JOIN portal_link_category c ON c.category_code = l.category_code
JOIN portal_link_group g ON g.group_code = c.group_code
ORDER BY g.sort_order, c.sort_order, l.sort_order;
```

## 3. 規則與注意事項（重要）

1. **只維護這 4 張 `portal_*` 表**（＋公告）。**不要用 SQL 去改業務資料表**（如 `rent_confirm`、`room`、`site`、`tenant_contract`、`billing_run` …）——那些請一律透過系統畫面操作，直接改 SQL 可能破壞資料一致性。
2. **代碼要對得上**：`portal_link.category_code` 必須等於某個 `portal_link_category.category_code`；`portal_link_category.group_code` 必須等於某個 `portal_link_group.group_code`。對不上的連結不會顯示。
3. **「隱藏」優先用 `is_enabled = false`**（可逆、安全）。真的要刪再用 `DELETE`。
4. **連結 url**：站內頁用相對路徑（`/master/site`、`/billing`、`/data`…）；外部網站用完整網址（`https://...`）。
5. 改完務必**重新整理瀏覽器**確認；若沒變，多半是代碼對不上或 `is_enabled=false`。
6. 目前站內可用的原生頁：`/master/site`、`/master/meter`、`/master/room`、`/upload/price`、`/upload/reading`、`/billing`、`/data`、`/audit`。

## 4. 建議：給 PM 一個「只能改 portal」的專用帳號

若要讓 PM 自行維護又不怕誤動業務資料，建議請系統管理者（超級使用者）在資料庫建一個**只對 portal_* 表有寫入權限**的角色。需要時再請工程師協助設定。
